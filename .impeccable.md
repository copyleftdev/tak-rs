# tak-rs — Design Context

## Users

- **Primary**: Senior systems engineers and security operators evaluating tak-rs as a replacement for the upstream Java TAK Server. They've run perf tools, they've read invariant docs, they're skeptical of marketing claims.
- **Secondary**: Defense / first-responder TAK ecosystem audience — situational awareness, military, search-and-rescue.
- **Context**: Read at a desk, often in a dark office. Will scroll fast on first visit, scan numbers and architecture before reading prose. Click GitHub if convinced.

## Brand Personality (3 words)

**Mil-spec · measured · uncompromising.**

The tone is institutional gravitas, not fashion. The kind of voice you'd find in a NATO field manual, an SR-71 pilot's checklist, or a rust-lang RFC — flat declarative sentences, no exclamation marks, real numbers with provenance. Confidence by demonstration, not by adjective.

## Aesthetic Direction

- **Theme: dark.** The audience reads at a desk in a dark office. Dark also fits the operator/terminal heritage. This is contextual, not "to look cool."
- **Palette: locked to brand tokens.** Three colors carry visual weight (60-30-10):
  - 60% surface: ink (`#0B1014`)
  - 30% structure: bone (`#F2EFEA`) for primary type, graphite (`#5A6470`) for secondary
  - 10% accent: rust (`#CE422B`) — used SPARINGLY, never as a flourish
- **Typography**: three-tier system. Display: **Archivo Black** (industrial geometric grotesque, aerospace label heritage). Body: **Atkinson Hyperlegible** (distinctive, accessibility-first, not in any reflex list). Mono: ui-monospace stack — used only for the brand wordmark and tabular numbers, NOT as generic "tech" decoration.
- **References**: 1970s NASA mission patches, Massimo Vignelli's NYC Subway map, Klim Type's Domaine Sans specimen, Linear's marketing site precision. Anti-references: any glowing-cyan-on-dark dashboard, any startup with neon gradients, any "AI" landing page from 2024-2025.

## Design Principles

1. **Asymmetric, intentional layouts.** No identical card grids. No metric-tile templates. Numbers earn their position through hierarchy, not standardization.
2. **One asymmetric accent per composition.** The brand mark says "the target is Rust" with one orange dot. The site echoes this: rust orange appears only where it carries meaning (a target dot, a reading anchor, the CTA tip), never as decoration.
3. **Proof over claim.** Every metric carries provenance ("5-min soak, 50 conns × 200 Hz"). The visitor can replicate it. No "blazingly fast" — show the µs.
4. **Motion serves narrative.** Page-load is a system boot sequence (mark draws in stroke-by-stroke, wordmark resolves, status row settles). Scroll-driven counter ticks reveal numbers. CTA hover converges particles into the mark — kinetic effects always reinforce "the target is Rust." Reduced-motion: all motion off, mesh static.
5. **No frameworks.** The site is one HTML file, ~1.5k lines, vanilla JS canvas + IntersectionObserver. The technical sensibility of the project is the technical sensibility of the site.
